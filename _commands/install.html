engineer install

Install Engineer’s hidden tracking lists in SharePoint.

The install command creates two lists at the root of the SharePoint site (URL configured in env.js) if they do not already exist. The first list is called EngineerMigrations , and is used to store the migration status for all Engineer migrations. The second list is called EngineerManifest, and is used to store content type IDs for any custom content types created by Engineer. These content types can later be referenced by name, allowing Engineer to automatically fetch the correct content type ID, even if they differ from environment to environment.

List Visibility

Engineer lists are hidden and are configured to be excluded from all search indicies, ensuring end-users never see these lists or the data contained therein.

Examples

{% highlight sh %} engineer install {% endhighlight %}
