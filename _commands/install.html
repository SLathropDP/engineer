--- title: install ---

engineer install

Install Engineer’s hidden tracking lists in SharePoint.

The install command creates two lists at the root of the SharePoint site if they do not already exist. The first list is called EngineerMigrations , and is used to store the migration status for all Engineer migrations. The second list is called EngineerManifest, and is used to store content type IDs for any custom content types created by Engineer. These content types can later be referenced by name, allowing Engineer to automatically fetch the correct content type ID, even if it differs from environment to environment.

Examples

{% highlight sh %} engineer install {% endhighlight %}

List Visibility

Engineer lists are hidden and configured to be excluded from all search indicies, ensuring end-users never see these lists or the data contained therein. You are free to further alter the permissions for these lists after installation. Use the browse command to navigate to these lists after installation.
