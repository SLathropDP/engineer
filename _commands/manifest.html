--- title: manifest ---

engineer manifest

Show the content type ID manifest.

The manifest command will retrieve manifest data from Engineer’s manifest tracking list and display it in a table in the console.

Examples

{% highlight sh %} engineer manifest {% endhighlight %}

Why use a manifest?

Content type IDs can be a bit unruly because of their length and structure. To make these easier to work with, Engineer uses a content type ID manifest list, which is used to store content type IDs for any custom content types created by migrations. These content types can later be referenced by name, allowing Engineer to automatically fetch the correct content type ID even if it differs from environment to environment.
