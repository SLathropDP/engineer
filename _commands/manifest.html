--- title: manifest ---

engineer manifest

Show the content type ID manifest.

The manifest command will retrieve manifest data from Engineer’s manifest tracking list and display it in a table in the console.

Examples

{% highlight sh %} engineer manifest {% endhighlight %}

Why use a content type ID manifest?

As of this writing, SharePoint honors field IDs when created using the REST API. It will not, however, honor content type IDs when created using the REST API. Any content type ID that you provide when creating a new content type will be ignored by SharePoint. For this reason, Engineer uses a manifest list to store content type IDs for future reference. Always use the convenience method (contentTypes.getByName('My Content Type')) when referencing a custom content type created by Engineer, and avoid referencing any custom content types by their ID (which could change from environment to environment).
