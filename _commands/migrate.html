--- title: migrate ---

engineer migrate [options]

Run pending migrations.

Options

-t, --to <file>
Run all migrations up to and including the specified file (optional)
-o, --only <file>
Run only the specified migration file, ignoring all other pending migrations (optional)
-s, --step <number>
Run pending migrations in order, but stop migrating after the maximum number of migrations (optional)
-f, --force
Run migrations, even if they have already been migrated (optional)

The migrate command deploys pending migrations to the target SharePoint environment. Migrations are run in chronological order (oldest to newest), based on their timestamp file names.

Examples

{% highlight sh %} # Run all pending migrations engineer migrate # Run pending migrations up to a specific migration engineer migrate -t 2017102612010101-update-fields # Run a specific migration engineer migrate -o 2017102612010101-update-fields # Run the next two migrations engineer migrate -s 2 # Run a specific migration in force mode (run it again) engineer migrate -o 2017102612010101-update-fields -f {% endhighlight %}

About Migrations

Migrations include both up (deployment) and down (retraction) methods. The migrate command executes only the up method of each pending migration. Learn more about migrations in the Migrations API documentation.
