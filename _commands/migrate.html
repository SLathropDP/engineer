engineer migrate [options]

Run pending migrations.

Options

  • -t, --to <file> — run all migrations up to and including the specified file
  • -o, --only <file> — run only the specified migration file, ignoring all other pending migrations
  • -s, --step <number> — run pending migrations in order, but stop migrating after the maximum number of migrations
  • -f, --force — run migrations, even if they have already been migrated

The migrate command will activate all pending migrations and run them in order on the target SharePoint environment. Migrations are run in chronological order (oldest to newest), based on their timestamp file names.

Migrations include both up (deployment) and down (retraction) methods. The migrate command executes only the up method of each pending migration.

Examples

{% highlight sh %} # Run all pending migrations engineer migrate # Run pending migrations up to a specific migration engineer migrate -t 2017102612010101-update-fields # Run a specific migration engineer migrate -o 2017102612010101-update-fields # Run the next two migrations engineer migrate -s 2 # Run a specific migration in FORCE mode (run it again) engineer migrate -o 2017102612010101-update-fields -f {% endhighlight %}
