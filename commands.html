Global Configuration

There are several global configuration options in Engineer that apply to all commands.

-c, --config <file>

Use -c <file> or --config <file> with any command to override the default env.js configuration file name. This allows you to use multiple environment files with the same set of migrations.

Options

  • file — the path to the configuration file you would like to use, relative to the current working directory (required)

Examples

{% highlight sh %} # Check the status of the environment configured in env-dev.js engineer -c env-dev.js status # Check the status of the environment configured in env/prod.js engineer -c env/prod.js status {% endhighlight %}

-q, --quiet

Use -q or --quiet to silence all logging and output (including errors) from Engineer. This may be useful in a scenario where migrations are being automated and no logging is necessary.

Examples

{% highlight sh %} # Run migrations in quiet mode engineer -q migrate # Run rollbacks in quiet mode engineer -q rollback {% endhighlight %}

-i, --info

Use -i or --info to increase the logging level and see more information about every request made by Engineer, including each request URI and response summary.

Examples

{% highlight sh %} # Run migrations in info mode engineer -i migrate # Run rollbacks in info mode engineer -i rollback {% endhighlight %}

-v, --verbose

Use -v or --verbose to show maximum logging. Note that this generates a lot of output. You have been warned!

Examples

{% highlight sh %} # Run migrations in verbose mode engineer -v migrate # Run rollbacks in verbose mode engineer -v rollback {% endhighlight %}
