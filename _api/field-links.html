--- title: FieldLinks ---

FieldLinks

Configure fields within a content type.

Access

Access the FieldLinks API using the fieldLinks property of the ContentType API.

Examples

{% highlight js %} up(engineer) { // Add field to web content type engineer.web.contentTypes.getByName('My Content Type').fieldLinks.add('My Field'); // Add field to list content type engineer.web.lists.getByTitle('My List').contentTypes.getByName('My Content Type').fieldLinks.add('My Field'); } {% endhighlight %}

Methods

add(fieldName)

Add an existing field to the content type. This method uses the SharePoint client-side object model (CSOM).

Parameters

fieldName
The name of the existing field or column to be added to the content type; this field must already be available on the parent Web or List (String, required)

Examples

{% highlight js %} up(engineer) { // Create fields engineer.web.fields.add('Breed'); engineer.web.fields.add('Weight'); // Create content type engineer.web.contentTypes.add('Pet'); // Add fields to content type engineer.web.contentTypes.getByName('Pet').fieldLinks.add('Breed'); engineer.web.contentTypes.getByName('Pet').fieldLinks.add('Weight'); } {% endhighlight %}

remove(fieldName)

Remove a field from the content type. This method uses the SharePoint client-side object model (CSOM).

Parameters

fieldName
The name of the field or column to be removed from the content type (String, required)

Examples

{% highlight js %} down(engineer) { // Remove fields from content type engineer.web.contentTypes.getByName('Pet').fieldLinks.remove('Breed'); engineer.web.contentTypes.getByName('Pet').fieldLinks.remove('Weight'); } {% endhighlight %}
